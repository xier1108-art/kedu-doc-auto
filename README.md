# K-에듀파인 비전자문서등록 자동 입력기

PDF 한 개를 드래그하면 Claude Vision AI 가 문서 내용을 분석해서 K-에듀파인 비전자문서등록 폼을 자동으로 채워주는 도구.

> **현재 버전**: v1.0.7 (2026.05 기준) — 시작 시간 ~2초, GUI 단순화, 모델 동적 선택

### 처음 사용 시 흐름

```
1. 가이드(docx) 다운로드 → Anthropic API 키 발급 (~10분, 가이드 따라하면 쉬움)
2. ZIP 다운로드 → 압축 풀기 → 폴더 안의 비전자문서등록.exe 더블클릭
3. [⚙ 설정] 클릭 → API 키 붙여넣기 → 저장
4. K-에듀파인 비전자문서등록 폼 열고 본인 정보 1회 캡처:
   - 접수자/업무담당자 옆 🔍 → 본인 선택 → GUI [👤 사용자 저장]
   - 과제카드 옆 🔍 → 카드 선택 → GUI [📌 과제카드 저장]
5. PDF 드래그 → [✅ 폼에 입력] → 자동 입력 → 검토 후 [저장]
```

## 🎯 동작 흐름

```
PDF (스캔 / 디지털 모두 지원)
   │
   ↓ Claude Vision AI (OCR + 의미 분석)
   ↓
폼 자동 입력:
   ● 기본값 (등록구분, 비공개, 5호, 기관, 설정안함 등) — DEFAULTS 상수
   ● 사용자 정보 (접수자/업무담당자, 부서) — 1회 캡처값 복원
   ● 과제카드 — 1회 캡처값 복원
   ● PDF 메타 (제목, 발신, 수신, 문서번호, 시행일자, 쪽수 등)
   │
   ↓ 화면에서 검토
   ↓
[저장] 버튼은 사람이 직접 클릭
```

## 📄 지원하는 PDF 유형

자동으로 두 가지 양식을 식별:

### A 유형 — 일반 공문
- 학교/공공기관/시공사/설계사 공문
- 패턴: "수신:", "참조:", "제목:", "시행 ○○제2026-NN호" 줄
- 예: `26.04.28 대성(갈현초)제2026-84호. 설계도서 검토서.pdf`
- 예: 장위토건 기성청구서 (사기업 공문)

### B 유형 — 조달청 검사검수요청서
- 나라장터 전자문서
- 패턴: "검 사 검 수 요 청 서" 헤더 + "[공공기관]/[납품업체]" 박스
- 추출 시 제목은 `검사검수요청서[계약건명]` 형식, 발신은 [납품업체] 의 회사명

## 📋 자동 입력 항목

### 기본값 (항상 적용 — DEFAULTS 상수에서 변경 가능)

| 항목 | 값 |
|---|---|
| 등록구분 | 접수 |
| 대국민공개여부 | 비공개 |
| 공개제한근거 | 5호 |
| 직원열람범위 | 기관 |
| 직원열람제한 | 설정안함 |

### 1회 캡처 후 복원

| 항목 | 캡처 방법 |
|---|---|
| 접수자 / 업무담당자 + 부서 ID | [⚙ → 👤 사용자 저장] |
| 과제카드 | [📌 과제카드 저장] |

### PDF 에서 AI 가 자동 추출

| 항목 | 폼 ID |
|---|---|
| 제목 | `TITLE` |
| 발신기관 / 발신명의 / 원기안부서 | `DRFTINSTTNM` / `DSPTCHNMCN` / `ORGDRAFTDEPTNAME` |
| 수신자 | `RECEIVERNAME` |
| 문서번호 | `DOCREGNO` |
| 시행일자 | `ENFORCEDATE` |
| 쪽수 | `PAGECNT` (PDF 페이지 수 자동) |
| 본문 요지 | `docOutline` |

## 🤖 Claude 모델 선택

[⚙ 설정] 다이얼로그에서 본인 키로 사용 가능한 모든 Anthropic 모델을 선택 가능. [🔄 새로고침] 으로 신규 모델 즉시 반영.

| 모델 | 정확도 | 비용 | 추천 용도 |
|---|---|---|---|
| **Claude Opus 4.7** | ★★★★★ | ~50원/PDF | 중요 문서, 정확도 최우선 |
| **Claude Sonnet 4.6 / 4.5** ★ 기본 | ★★★★ | ~10원/PDF | 일반 업무 (균형) |
| **Claude Haiku 4.5** | ★★★ | ~1원/PDF | 디지털 PDF 위주, 비용 절감 |

* 기본값: 본인 키로 사용 가능한 최신 Sonnet

## ⚠ 주의사항

1. **[저장] 은 자동으로 누르지 않습니다.** 자동 입력 후 화면에서 검토 → 사람이 직접 클릭.
2. **첨부파일은 수동**: K-에듀파인 [파일추가] 버튼을 직접 클릭해서 업로드 필요 (보안상 JS 자동화 불가).
3. **사용자/과제카드는 1회 캡처 필요**: K-에듀파인이 검색 팝업으로만 입력 가능한 항목이라 1회 수동 선택 후 GUI 저장.
4. **OCR 오인식 가능**: 한국어 PDF 의 글자 오인식이 종종 발생 (인명/회사명 등). 결과는 항상 사람이 검토.
5. **PDF 페이지 제한**: Claude API 의 100페이지 제한 회피를 위해 자동으로 첫 5페이지만 사용 (표지는 어차피 1~2페이지에 있음).

## 📂 파일 구성

| 파일 | 역할 |
|---|---|
| `gui.py` | 메인 GUI (드래그앤드롭, 추출 결과 표시, 사용자/과제카드 저장 버튼) |

| `splash.png` | 시작 스플래시 이미지 |
| `guide/` | API 키 발급 가이드 (docx + 실제 스크린샷) |

## 🛠 개발자용 — 빌드 / 배포

### 의존성 설치
```cmd
pip install --user pywinauto comtypes pdfplumber pypdf anthropic tkinterdnd2 pywin32 pymupdf pillow python-docx pyinstaller
```

### 빌드
```cmd
python build.py            # onefile EXE (단일 파일, 첫 실행 30~60초)
python build.py --onedir   # onedir ZIP (빠른 시작 ~2초, 권장)
python build.py --debug    # 콘솔 빌드 (에러 메시지 확인용)
```

### 배포
```cmd
# GitHub Release (gh CLI)
gh release create vX.Y.Z dist\kedu-doc-auto-vX.Y.Z.exe dist\kedu-doc-auto-vX.Y.Z-fast.zip \
    --title "vX.Y.Z" --notes "변경 내용..."
```
